# 컴퓨터 구조

CPU가 명령어를 어떻게 겹쳐 처리하고, 메모리 접근 비용을 어떻게 줄이는지 이해하는 주제입니다.

## 명령어 파이프라이닝

명령어 파이프라이닝은 CPU가 하나의 명령어 실행 과정을 여러 단계로 나누고, 서로 다른 명령어의 단계를 겹쳐 실행해 처리량을 높이는 기법입니다. 한 명령어가 모든 단계를 끝낼 때까지 다음 명령어를 기다리는 대신, 각 단계가 다른 명령어를 동시에 처리합니다.

![Five stages pipeline](https://upload.wikimedia.org/wikipedia/commons/thumb/2/21/Fivestagespipeline.png/960px-Fivestagespipeline.png)

## 파이프라인 위험

파이프라이닝은 이상적으로는 처리량을 높이지만, 명령어 간 의존성이나 자원 충돌 때문에 멈추거나 잘못 계산한 결과를 버려야 할 수 있습니다. 이런 상황을 파이프라인 위험이라고 합니다.

| 위험 | 설명 |
| --- | --- |
| 데이터 위험 | 뒤 명령어가 앞 명령어의 연산 결과를 필요로 하는데 아직 결과가 준비되지 않은 경우 |
| 제어 위험 | branch, 조건문 때문에 다음에 가져올 명령어를 확정하지 못하는 경우 |
| 구조적 위험 | 여러 명령어가 ALU, 레지스터, 메모리 포트 같은 같은 하드웨어 자원을 동시에 요구하는 경우 |

제어 위험은 분기 예측으로 완화할 수 있지만, 예측 실패 시 이미 채운 파이프라인을 비우고 다시 시작해야 하므로 성능 손실이 발생합니다.

## 참조 지역성

참조 지역성은 CPU가 메모리에 접근할 때 보이는 반복적 경향입니다. 캐시 메모리는 이 경향을 이용해 메모리 접근 비용을 줄입니다.

| 종류 | 의미 | 예시 |
| --- | --- | --- |
| 시간 지역성 | 최근 접근한 메모리에 다시 접근할 가능성이 높음 | 지역 변수나 반복문에서 같은 값을 재사용 |
| 공간 지역성 | 접근한 메모리 근처에 접근할 가능성이 높음 | 배열을 앞에서 뒤로 순차 순회 |

## Java 2차원 배열과 공간 지역성

Java의 `int[][]`는 하나의 연속된 2차원 메모리 블록이 아니라 `int[]` 객체들에 대한 참조 배열입니다. `array[i]`는 각각 독립된 `int[]` 객체이며, 메모리상 연속 배치가 보장되지 않습니다.

열 우선 순회는 서로 다른 행 객체를 계속 건너뛰며 접근하므로 캐시 히트율이 떨어질 수 있습니다.

```java
for (int j = 0; j < size; j++) {
    for (int i = 0; i < size; i++) {
        array[i][j]++;
    }
}
```

행 우선 순회는 같은 `int[]` 내부를 연속적으로 접근하므로 공간 지역성을 더 잘 활용합니다.

```java
for (int i = 0; i < size; i++) {
    for (int j = 0; j < size; j++) {
        array[i][j]++;
    }
}
```

핵심은 "캐시는 인접한 데이터를 함께 가져온다"는 점입니다. 배열, 버퍼, 파일, 디스크 접근 모두 순차 접근이 랜덤 접근보다 유리한 이유를 이 관점으로 설명할 수 있습니다.
